# Spring

```text
核心功能模块：
    1。控制反转容器IOC（依赖注入）
    2。面向切面编程（AOP）
    3。处理访问（DAO层支持）
    4。事务管理
    5。MVC
    6。远程访问
    。。。。
    
    
1。把创建对象的权利交给框架，也就是指将对象的创建，对象的存储，
    对象的管理交给了spring容器。spring容器是spring中的一个
    核心模块，用于管理对象，底层可以理解为是一个map集合
2。AOP就是将那些与业务无关，却为业务模块所共同调用的逻辑或责任
    分开封装起来，便于减少系统的重复代码，降低模块间的耦合度，
    利于可操作性和可维护性
3。Spring Data实现了对数据访问接口的统一，支持多种数据库访问
    框架或组件（如：JDBC，Hibernate，MyBatis）作为最终数据
    访问实现
4。Spring框架为事务管理提供了一致的抽象，具有以下优点：跨不同
    事务的API，支持声明式事务，
5。Spring MVC实现了基于MVC的设计方法的实现，结合基于Java注解
    的配置，允许开发者开发出低代码侵入的Web应用项目，并简便的实现
    大部分Web功能
6。“约定大于配置“的快速应用开发，Spring Boot    
```
